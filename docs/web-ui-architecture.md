# Web UI 技术架构

## 目标

- 提升可维护性：将纯逻辑与界面渲染分离
- 提升性能：控制首页请求到可用状态在 2 秒以内
- 提升体验：统一视觉、响应式布局、键盘与屏幕阅读器友好

## 分层设计

- `web_ui.py`
  - 页面路由与组件编排
  - 下载流程状态与交互控制
  - 登录页与鉴权中间件挂载
- `fanbox_extractor/web_ui_core.py`
  - URL 安全构建
  - Markdown 链接重写
  - 文件树构建
  - 文件大小格式化
  - 鉴权辅助函数

## 性能策略

- 首页只渲染核心信息，复杂内容按标签页与交互触发
- 文件树按需刷新，避免无意义频繁扫描
- UI 样式集中注入，减少零散样式拼接与重复计算
- 通过 E2E 冒烟测试校验首页响应时间阈值

## 响应式策略

- 断点布局
  - 桌面：文件树与预览区并排
  - 平板/移动：自动堆叠，优先展示核心操作
- 按钮与输入框支持自适应宽度
- 日志区在移动端降低默认高度

## 无障碍策略（WCAG 2.1 AA 对齐）

- 增加跳转主内容链接
- 关键控件增加 `aria-label`
- 状态信息使用 `aria-live=polite`
- 加强键盘焦点可视化
- 支持 `prefers-reduced-motion`，降低动画干扰

## 浏览器兼容策略

- 基线：Chrome、Firefox、Safari(WebKit)、Edge(Chromium)
- 功能验证覆盖：
  - 登录重定向
  - 主界面渲染
  - 文件树展开与预览
  - 下载按钮行为
